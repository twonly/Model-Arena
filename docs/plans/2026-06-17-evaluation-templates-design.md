# 评测模板页设计确认稿

## 当前结论

评测模板页第一阶段建议采用“卡片列表 + 右侧预览抽屉 + 直接试跑”的结构，不直接把每个卡片都做成详情页。

原因很直接：模板页的首要任务是帮用户快速判断“这个 Prompt 是否值得我跑一轮”。卡片点击展开预览比进入详情页更轻，用户能在同一页连续浏览、对比、试跑。详情页更适合 SEO、分享和沉淀长期样本，但每一次点击成本更高，第一阶段只给官方高价值模板开放详情页更稳。

示意稿位于：

- `docs/mockups/evaluation-templates-page.html`

## 页面结构

- 顶部：品牌、排行榜、趋势、账号入口，保持 TOKRACE 现有工具型风格。
- 标题区：说明“评测模板”是可复用的 Prompt 评测集。
- 分类 Tabs：官方经典、我的模板、速度与 token、事实准确、长文输出、视觉理解、众测中。
- 主区域：模板卡片网格。
- 右侧：当前选中模板的 Prompt 预览、关键字段、众测结果预览与行动按钮。
- 底部说明：交互取舍与众测策略。

## 卡片信息

每张卡片展示足够多的信息，让用户不打开详情也能判断是否要跑：

- 来源：官方 / 用户模板 / 私有模板
- 分类：速度基准、事实准确、长文能力、视觉理解等
- 预计耗时：约 2 分钟 / 4 分钟 / 6 分钟
- 标题与一句话用途
- 标签：TTFT、TPS、Token 消耗、幻觉风险、字数达成等
- 样本数
- 样本可信度：低 / 中 / 高
- 推荐模型数
- 主按钮：立即试跑
- 次按钮：预览 Prompt / 编辑

## 点击展开还是进入详情页

推荐规则：

- 卡片点击：展开右侧预览抽屉。
- `立即试跑`：直接把模板 Prompt 带入 `/arena?template=...` 或 session seed。
- `进入详情页`：只在官方高价值模板、已有足够样本或具备 SEO 搜索价值时展示。

取舍：

- 只展开：浏览快、成本低、适合工具流，但 SEO 承接弱。
- 只进详情页：SEO 和分享强，但用户浏览摩擦高。
- 混合方案：默认展开，少数模板有详情页，最适合当前阶段。

## 用户自定义模板

当前 `/arena` 已有 Prompt 库和自定义 Prompt。模板页应复用这套数据：

- 本地私有模板默认只存在用户浏览器。
- 用户可添加标题、分类、标签、Prompt、评分维度。
- 用户主动公开后，才进入公开模板池。
- 导入/导出能力沿用现有 Prompt 库，避免重复造一套存储。

## 是否支持直接试跑

需要支持。模板页不只是展示页，它应该是一个高意图入口。

第一阶段交互：

1. 用户点击“立即试跑”。
2. 系统把模板标题、说明、Prompt 作为临时草稿带入 `/arena`。
3. 若用户已有本地模型配置，默认使用其专业模式模型。
4. 若没有配置，可使用快速样例模型，但要明确会消耗免费额度。
5. 跑完后可生成分享链接、长图和评测稿。

## 跑完后是否记录在该模板下面

建议支持，但必须用户确认后才公开。

默认逻辑：

- 跑完后本地自动进历史记录。
- 若该 Prompt 来自模板，历史记录带上 `templateId`。
- 页面提示：“保存到该模板样本？”
- 用户确认后，只上传匿名指标、模型名、时间和可选评分。
- 不上传 Prompt 以外的新输入内容，不上传 API Key。

## 是否做众测

建议做轻量众测，不做复杂社区。

第一阶段只沉淀：

- 每个模板下的模型速度分布
- 首 Token 中位数
- 输出 TPS 中位数
- token 消耗均值
- 样本数与可信度标签
- 用户可选人工评分：事实准确、完整度、Prompt 遵循度、性价比

不做：

- 大型评论社区
- 复杂声望系统
- 自动 AI 裁判总榜
- 付费任务广场

这样既能让模板页有“众测”价值，又不会把产品重心从评测工具拖向社区运营。

## 待确认点

1. 模板页正式路径用 `/templates` 还是 `/prompts`。
2. 第一批官方模板数量：建议 8-12 个。
3. 是否允许用户公开自己的模板。
4. 跑完后保存到模板样本是否默认弹一次确认。
5. 哪些模板值得独立详情页和 SEO 页面。
